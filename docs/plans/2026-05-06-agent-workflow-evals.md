# Agent Workflow Evals Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Turn Minty's agent-native product direction into a repeatable synthetic eval suite that proves MCP/CLI answers stay source-backed, redacted, freshness-aware, and honest across core Hermes/OpenClaw workflows.

**Architecture:** Extend the existing deterministic evaluation harness instead of adding an LLM judge. Keep `crm/evaluation.js` as the pure assertion engine, teach `scripts/evaluate-network-memory.js` to run both CLI retrieval and MCP tool scenarios against `data-demo`, add reusable synthetic workflow cases under `tests/fixtures/agent-workflows.json`, and wire a focused `npm run network:evaluate` check. Evals assert machine-readable contracts: expected tool, minimum/maximum results, required evidence kinds, required safety fields, forbidden substrings/fields, low-evidence empty states, and source-freshness warnings.

**Tech Stack:** Plain Node.js CommonJS, Node built-in test runner, existing `crm/evaluation.js`, `scripts/evaluate-network-memory.js`, `scripts/minty-mcp-server.js`, `scripts/agent-query.js`, `scripts/seed-dev-data.js`, `tests/unit/evaluation.test.js`, `package.json`, `data-demo` generated by `npm run seed:demo`.

---

## Product framing

Minty is now pointed at AI-native private network memory. Recent plans cover citations, source health, Hermes readiness, meeting prep, intro paths, goal next actions, and the UI as an evidence-review workbench. The remaining strategy gap is not another tool: it is a way to prevent the agent surface from quietly drifting back into vibes, CRM busywork, or privacy leaks as builders add tools.

The repo already has a small `network:evaluate` harness, but it only checks a few `queryNetwork()` cases. Hermes/OpenClaw value depends on workflows, not just one query function: source-specific searches should respect source health, impossible queries should fail empty, workflow briefs should expose freshness and citations, and MCP envelopes should keep contact details out. This plan upgrades evals into a product-contract gate for the dogfood loop.

This complements existing plans:

- `2026-05-02-agent-retrieval-citations.md` defines evidence/citation fields per recommendation.
- `2026-05-06-agent-source-health-mcp.md` defines source readiness preflight.
- `2026-05-06-evidence-review-workbench.md` gives humans a correction loop.
- This plan makes those promises regression-testable with synthetic cases before Sree trusts Minty in Hermes.

## Success criteria

- `npm run network:evaluate` seeds/uses demo data and exits non-zero when any agent workflow violates the contract.
- Eval cases can target `query_network`, `mcp:search_network`, `mcp:person_context`, `mcp:workflow_brief`, and, once implemented, `mcp:source_health` / `mcp:goal_next_actions` without needing separate scripts.
- Evals assert redaction by checking both forbidden fields and forbidden fixture substrings.
- Evals assert honesty: impossible/private-codename queries must return no results or low-confidence empty states, never fallback-only recommendations.
- Evals assert trust metadata: source-backed workflows include evidence/citations/confidence/freshness/safety fields appropriate to the tool.
- The suite uses synthetic fixtures only and never reads or snapshots real `data/`.

## Non-goals

- Do not add an LLM-as-judge, embeddings, new npm dependencies, or network calls.
- Do not evaluate subjective prose quality in this plan; evaluate deterministic contract shape and privacy/trust guarantees.
- Do not add real Sree contact data, raw message bodies, emails, phones, token paths, or private strategy notes to fixtures.
- Do not block all feature work on exhaustive evals. Start with a small high-signal suite that catches privacy and hallucination regressions.
- Do not replace unit tests; evals sit above them as workflow-level contract checks.

**Step 3: Write minimal implementation**

```js
function pathExists(obj, dottedPath) {
    const parts = String(dottedPath || '').split('.').filter(Boolean);
    let cur = obj;
    for (const part of parts) {
        if (cur == null) return false;
        const key = Array.isArray(cur) && /^\d+$/.test(part) ? Number(part) : part;
        if (!Object.prototype.hasOwnProperty.call(Object(cur), key)) return false;
        cur = cur[key];
    }
    return cur !== undefined && cur !== null;
}

function serializedContains(output, needle) {
    if (!needle) return false;
    return JSON.stringify(output).includes(String(needle));
}
```

Then inside `evaluateOne()` after existing evidence checks:

```js
for (const required of Array.isArray(testCase.requirePaths) ? testCase.requirePaths : []) {
    if (!pathExists(output, required)) failures.push(`missing_required_path:${required}`);
}
for (const forbidden of Array.isArray(testCase.forbidPaths) ? testCase.forbidPaths : []) {
    if (pathExists(output, forbidden)) failures.push(`forbidden_path_present:${forbidden}`);
}
for (const forbidden of Array.isArray(testCase.forbidSubstrings) ? testCase.forbidSubstrings : []) {
    if (serializedContains(output, forbidden)) failures.push('forbidden_substring_present');
}
if (testCase.expectEmpty === true && results.length !== 0) failures.push('expected_empty_results');
```

Keep the return shape backward-compatible.

```js
const { handleMessage } = require('./minty-mcp-server');
```

3. Add a runner:

```js
function runMcpTool(target, args, data) {
    const name = target.replace(/^mcp:/, '');
    const response = handleMessage({
        jsonrpc: '2.0',
        id: 1,
        method: 'tools/call',
        params: { name, arguments: args || {} },
    }, data);
    if (response.error) return { error: response.error, results: [] };
    const text = response.result && response.result.content && response.result.content[0] && response.result.content[0].text;
    try { return JSON.parse(text); }
    catch { return { error: 'invalid_mcp_json', results: [] }; }
}
```

4. Dispatch by target:

```js
const report = evaluateRelationshipQueries(cases, (query, testCase) => {
    if (String(testCase.target || '').startsWith('mcp:')) return runMcpTool(testCase.target, testCase.arguments, data);
    return queryNetwork(query, { ...data, limit: 10 });
});
```

5. Update `crm/evaluation.js` so `evaluateRelationshipQueries()` passes the whole case to the callback:

Privacy checks:

- Eval fixtures contain no real emails, phones, raw contact IDs, message bodies, private Minty strategy, or Sree-specific relationship data.
- Evals fail if MCP/CLI envelopes include direct contact fields (`email`, `phone`, `emails`, `phones`, `rawContact`) or known synthetic secret strings.
- Evals fail if impossible queries return recommendations.
- Evals fail if expected safety/freshness/evidence paths are missing.
